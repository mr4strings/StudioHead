# Landing Page Requirements

This document outlines the requirements for the initial landing page of the game.

## General
- **Display**: The game must open in full screen mode.
- **Background**: 
  - The landing page features a static image as the background.
  - **Recommendation**: 
    - **Resolution**: 1920x1080 (Full HD) is the standard baseline. For higher fidelity support, consider 2560x1440 (QHD) or providing a scalable layout.
    - **Format**: `PNG` for lossless quality (crisp lines, text) or `JPG` (high quality, 85-90%) if file size is a concern. `WebP` is also an excellent modern alternative if the engine supports it.

## UI Elements (Buttons)
The landing page must contain the following interactive buttons:

1.  **New Game**
    - **Action**: Starts the new game workflow.

2.  **Open Game**
    - **Action**: Opens a page displaying the user's saved games for selection.

3.  **Database**
    - **Action**: Opens the Database management workflow.
    - **Purpose**: Allows users to modify the database to load different individuals, production companies, etc.

4.  **Editor**
    - **Action**: Launches the database editor tool.

5.  **Credits**
    - **Action**: Opens the Credits page.

6. **Options**
    - **Action**: Opens the Options page.
